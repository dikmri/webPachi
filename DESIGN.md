# webPachi 設計書

GitHub Pages で遊べる物理演算パチンコ台シミュレーター。
機種はオリジナルの海テーマ甘デジ「P ドリームオーシャン 99ver.」
(スペック・釘構成は一般的な羽根なしデジパチ/甘デジの典型構成を参考にしたオリジナル)。

## 技術スタック

- bun + Vite + TypeScript (strict) + matter-js (物理)
- 描画: Canvas 2D (盤面) + DOM/CSS (液晶・データカウンター・枠)
- 音: WebAudio API による手続き生成のみ (外部アセット・既存楽曲は一切使わない)
- デプロイ: GitHub Actions → GitHub Pages

## イベントフロー

```
[board] --BoardEvent--> [main] --転送--> [game(GameLogic)]
[game]  --GameEvent---> [main] --配線--> [reels] [audio] [dataCounter] [board(役物開閉)]
[frame] --handlePower-> [main] --launch--> [board]
[main] が PlayerState/Stats を所有し賞球加算・差玉サンプリングを行う
```

- 賞球テーブルは `SPEC.payout`。main が BoardEvent を見て持ち玉に加算する。
- `GameLogic.attackerShouldOpen` / `denSupport` を main が毎フレーム見て
  `board.setAttackerOpen()` / `setDenchuOpen()` を呼ぶ (電チューはゲート通過で1.5s開放、
  電サポ中のみ。開放制御は board 側でなく main 側タイマーでよい)。

## 盤面仕様 (エージェントA)

論理座標 480x660。重力は matter-js デフォルト方向(下向き)。

- 玉: 半径 5.5、restitution≒0.4、friction 小、密度デフォルト。上限 60個(超えたら古い玉を消す)。
- 釘: 半径 2.0 の静的円、restitution≒0.5。
- 外枠: 左右壁・上部は円弧(中心 (240,330) 付近、半径≒310 の上半分)。
- 発射レール: 左端に幅約14の誘導路(外壁と内レールの間)。玉は左下 (18,640) 付近から
  power に応じた初速で上方へ発射され、円弧に沿って盤面上部へ入る。
  強いほど右奥(ブッコミ超え)まで飛ぶ。レール出口に戻り防止片。
- センター役物(液晶枠): 矩形 x:130..350, y:150..330 の静的ボディ。
  左横にワープ入口(通った玉はステージへ)。ステージ=液晶下の皿(y≒335)で玉が左右に転がり、
  中央から落ちるとヘソ上に落下(ヘソ入賞率高)。
- 風車(ふうしゃ): 左 (105,290)・右 (375,290)。回転する十字体(ピン留め)。玉を散らす。
- 天釘: (228,108),(252,108) 付近に2本。
- 寄り釘/バラ釘: 左右打ち分け領域に計30本程度。
- 道釘: y≒560 に x:60..205 の水平列(間隔13〜14、玉が上を転がる)。途中1箇所こぼしの切れ目。
- ジャンプ釘→命釘: 道釘の終端で玉を跳ねさせ、命釘2本(中心間隔≒17)の間へ。
- ヘソ(スタートチャッカー): (240,558) 幅14のセンサー。
- 電チュー: ヘソ直下 (240,588)。閉時は入賞不可、開時は幅22の受け皿。
- スルーゲート: 左道中 (100,340)。通過検知のみ。
- アタッカー: (240,612) 幅56の開閉フラップ。開時は落下玉の大半を吸い込む。
- 一般入賞口: 左下に3個 (60,600)(85,614)(110,624) 付近。
- アウト口: 最下部 (240,648)。入賞しなかった玉を回収。
- 盤面背景: 海テーマ(青グラデ・泡・光条・砂地)を Canvas で手続き描画。「それっぽく」豪華に。

**調整目標(scripts/simulate.ts で検証)**: 適正ハンドル(power 0.55〜0.65 付近)で
1000発打って ヘソ入賞 55〜75発 (千円=250発あたり14〜19回転)。
シミュレーションは matter-js を固定タイムステップで回し、入賞内訳を表で出力する。

## ゲームロジック仕様 (エージェントB)

- 保留最大4。ヘソ入賞で保留+1(満タン時は無効、賞球のみ)。
- 変動時間: 通常時 基本8000ms / 保留3以上で4000ms / 電サポ中2000ms。
  リーチ時 +6000ms(ノーマル) or +14000ms(SP)。当たり時は70%がSP経由。
  ハズレ時のリーチ発生率: 通常時8% (うちSP 30%)。
- 図柄: 1..9 の3桁。確変当たり=奇数ゾロ目、通常当たり=偶数ゾロ目。
  16R確変は 7 揃い優先。ハズレはゾロ目にならない組合せ(リーチ時は左右一致・中±1)。
- 大当たり: ラウンドは attacker 入賞9個 or 9000ms で終了、インターバル1200ms。
- 通常当たり後: 時短100回。確変当たり後: 次回大当たりまで確変+電サポ。
- 時短中に当たった場合も振り分けは同一。時短を使い切ったら通常へ。
- 液晶(reels): DOM+CSSで3リール縦スクロール風。左→右→中の順に停止。
  リーチ時は中リールがスロー回転+「リーチ!」表示。大当たり中はラウンド数と
  「大当たり!」「確変突入!」等の演出表示。海テーマの配色。

## UI仕様 (エージェントC)

実際のパチンコホールにあるデータカウンター(呼び出しランプ)を忠実に再現する:

- 黒筐体 + 7セグ風表示。項目: 「大当り」(本日合計・赤)、「確変」(オレンジ)、
  「スタート」(現在回転数・緑)、「総スタート」、「前回」(前回当たりの回転数)。
- 大当たり履歴リスト(直近10件: 回転数/R数/確変か)。
- スランプグラフ(SlumpGraph): 黒背景に緑ライン(差玉プラス側)・赤(マイナス側)、
  0ライン・目盛り(±2500玉等)付き。パチンコ屋のデータ表示器風のデザインにする
  (一般的なチャート様式ではなく、実機データランプの様式を意図的に踏襲)。
- 台枠(frame.ts + CSS): 金色装飾枠・サイドランプ(大当たり中は虹色点滅)・上皿。
  上皿に持ち玉数(7セグ風)・投資額・「玉貸」ボタン(500円=125玉)・機種名プレート。
- ハンドル: 右下の円形ダイヤル。マウスドラッグ/タッチで回して開度0..1を調整、
  離しても保持(ストップボタンで0に)。キーボード←→でも微調整可。開度メーター表示。

## 音仕様 (エージェントD)

WebAudio のオシレーター/ノイズ合成のみで全SE・BGMを生成する(オリジナル曲。既存曲の模倣禁止):

- BGM: normal(のんびり海辺のマリンバ風ループ)、reach(緊張感のあるビート)、
  jackpot(明るいファンファーレ→アップテンポループ)、kakuhen(疾走感)、jitan(軽快)。
  8〜16小節のループをスケジューラで回す。切替はクロスフェード気味に。
- SE: types.ts の SeName 全種。nail は 80ms 以内の連打を間引く。
- ページ操作前は AudioContext を作らない(init() で解禁)。マスターゲインでミュート。

## 動作確認ループ

logger.ts の logger.log(category, message) を全モジュールの主要イベントで呼ぶこと
(発射・入賞・変動開始/停止・当たり・R進行・モード移行・UI操作・音切替)。
ユーザーは画面の「ログをダウンロード」ボタンでログを保存し、AIが解析する。
